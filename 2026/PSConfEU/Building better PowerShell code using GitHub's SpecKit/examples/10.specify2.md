/speckit-specify
The PSExcuse module currently has New-Excuse, which draws from a built-in set of excuses. Add the ability to extend that set at runtime.

Add a function Add-Excuse that lets a user contribute a new excuse to the module's corpus so future calls to New-Excuse can return it.

It supports:
- Text (required): the excuse text to add.
- Category (optional): a grouping such as Network, Hardware, Human, Cosmic. Defaults to a general/uncategorised group.

Behaviour and acceptance criteria:
- Added excuses persist in a module-managed data store so they survive across sessions.
- After Add-Excuse runs, the new excuse is eligible to be returned by New-Excuse.
- New-Excuse gains an optional Category parameter to restrict it to one category; with no Category it draws from the full combined set (built-in plus added).
- Adding a duplicate excuse does not create a second copy.
- Add-Excuse supports -WhatIf so the change can be previewed without writing it.
- The data store location is overridable (a parameter or module variable) so tests use a temporary location and stay isolated.
- Existing New-Excuse behaviour (Severity, Blame, Format) is unchanged for callers who don't use the new Category parameter.
