/speckit-specify
Add a feature in my module called New-Excuse that returns a believable, randomly chosen excuse for why a software deployment or task failed. By default it returns a single excuse as a string.

It supports three optional parameters:
- Severity: one of Minor, Moderate, Severe, CareerEnding. This shifts the tone of the excuse from a light hiccup up to a dramatic catastrophe. Default is Moderate.
- Blame: who or what the excuse blames — for example DNS, TheNetworkTeam, TheIntern, MercuryRetrograde, CosmicRays. When supplied, the excuse must reference that culprit. When omitted, a culprit is chosen at random.
- Format: how it's presented. Plain returns the bare excuse text. Slack wraps it in an apologetic stand-up-style update suitable for posting to a channel. Default is Plain.

Acceptance criteria:
- New-Excuse with no parameters returns a non-empty string.
- Severity visibly changes the tone (a CareerEnding excuse reads more dramatically than a Minor one).
- When Blame is specified, the chosen culprit appears in the output.
- When Format is Slack, the output includes a greeting and an apologetic framing, not just the raw excuse.
- Invalid Severity, Blame, or Format values are rejected with a clear error.
- With a fixed random seed, output is reproducible so behaviour can be tested deterministically.
